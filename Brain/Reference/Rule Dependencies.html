When a rule is changed, the crib consults this map to identify every other location that references the same icon, threshold, or concept, and updates each referenced location to match the changed rule. This file exists for that purpose only — a crib navigation aid, nothing more.

🔗 Shared-Concept Registry — One Rule, Many Files

Patterns defined once but used across multiple files. Change the concept = update every file listed.

  • Global parentheses ban
    Source: Icon Order and Format.html · locked 2026-05-17
    Used in: every section, every guide — no ( character in any visible text
    Validator-enforced. Affects Day Trips by Train headings (old format used parens for travel time — now Destination · travel time), stop descriptions, note lines, everywhere.
  • Tilde restriction
    Source: Icon Order and Format.html
    Used in: ⏰ average-time-spent rows only
    Validator-enforced. ~ is valid only immediately after ⏰. Banned in descriptions, addresses, Local Tastes entries, Day Trips by Train, notes, everywhere else. Previously appeared in Day Trips by Train intros and Local Tastes descriptions — both are fails.
  • Question mark ban
    Source: Icon Order and Format.html · locked 2026-05-17
    Used in: every section, every guide — no ? character in any visible text
    Validator-enforced. The guide format is declarative — it states facts, it does not ask questions. Any ? in visible text is a hard fail. Note: ? in URL query strings (HTML attribute values) is exempt — the check targets text nodes only.
  • Extras-sub heading URL rule
    Source: Icon Order and Format.html · Links.html
    Used in: every .extras-sub heading that links to a venue
    Validator-enforced. The href must point to a review platform: yelp.com, google.com/maps, maps.google.com, tripadvisor.com. Official venue-site URLs (restaurant's own domain, etc.) fail. Distinct from the 📍 locked-pattern check — that applies only to 📍 address rows.
  • Extras-sub heading suffix rule
    Source: Restaurants Near Hotel - Extra Section.html §5b · Downtown Restaurants - Extra Section.html §5 · Cappuccino - Extra Section.html §3 · locked 2026-05-24
    Used in: 🫕 Restaurants Near Hotel · 🍽️ Downtown Restaurants · ☕ Cappuccino — all three food/drink sections share the rule
    Validator-enforced (T4d for Cappuccino; RNH suffix check; DR suffix check — all added 2026-05-24). The .extras-sub heading line ends at the closing </a> of the review link. No text (cuisine style, em dash, extra label) may follow the anchor on that heading div. Cuisine and description belong on line 2 of .entry-body. Trailing text after </a> is a hard fail in all three sections.
  • Canonical section title enforcement
    Source: required_extras list in validate_itinerary.py · tightened 2026-05-24
    Used in: all 14 sections tracked by required_extras
    Validator-enforced. Section titles must match the exact canonical string — no legacy aliases accepted. Aliases removed 2026-05-24: "🫕 Restaurants Near Hotel Restaurants" alias removed (only "🫕 Restaurants Near Hotel" valid); bare "🍽️ Downtown" alias removed (only "🍽️ Downtown Restaurants" valid); bare "🎭 Shows" alias removed (only "🎭 Shows, Performances & Concerts" valid). Any drift from the exact title is now a hard fail.
  • No-underline rule
    Source: Links.html § 8 (re-homed 2026-06-06; originally Icon Order § 6, locked 2026-05-16, lost in a rewrite)
    Used in: all guide HTML — inline text-decoration:underline style overrides are banned
    Validator-enforced. Links use the stylesheet default; no element may set underline via inline style. A color change is the only visual marker of a clickable element. Applies guide-wide.
  • Seafood exclusion
    Source: Restaurants Near Hotel - Extra Section.html (about-me preference, restated in each affected section)
    Used in: Restaurants Near Hotel · Downtown Restaurants · Michelin (all aligned 2026-05-13)
    Rule: seafood-core venues excluded. Exception: sushi restaurants are permitted. Nothing else seafood-focused qualifies — salmon-focused, tuna-focused, oyster bars, fish houses, etc. are all out.
  • Negative-finding line pattern
    Pattern — no single owner, restated per section
    Used in: Cappuccino · Local Tastes · Food Delivery · Michelin · Pickleball · Stations · Shows · Day Trips by Train · Weekly Closures · Photos Rules (per-stop)

    Source: Motion Rule.html (extra section pattern)
