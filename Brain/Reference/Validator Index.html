✈️ GUIDES INDEX — FLIGHT-TIME VIEW

validate_flight_index.py runs independently of validate_itinerary.py — it audits the FMAP data block and the colour scheme of the "By flight time from Seattle" view in Travel-Website/Guides/guides_index.html. Schema + routing authority + colour rules are documented in a comment directly above var FMAP. Run after any FMAP or flight-view colour edit: python3 Brain/scripts/validate_flight_index.py (exit 0 = clean).

  • FMAP block present and parses as valid JSON
  • Every entry has the exact schema keys: t, m, r, d, i, h, rg, o (no missing, no extra)
  • Routing r is one of home | nonstop | seasonal | 1stop | 2stop
  • m is a non-negative integer; destination i is a 3-letter IATA
  • Connecting routings (1stop / 2stop) carry a hub IATA in h; nonstop / seasonal / home have h = null
  • Mosaic coverage — every guide card has an FMAP entry, and every FMAP entry (except home) has a mosaic card (no orphans)
  • ROUTING AUTHORITY — r agrees with Trip-Essentials/Delta-Routes-SEA.html tiers for any IATA listed there (t0 = nonstop/seasonal, t1 = 1stop, t2 = 2stop; best/fewest-stop tier wins when an IATA appears in several)
  • COLOUR CONSISTENCY — for nonstop / 1stop / 2stop the dot (CSS .fdot), the card border (JS COL), and the Stops filter-chip emoji all resolve to the same colour family (catches the "two-tone" bug; exact hexes may change freely)
  • Label t equals the displayed time (m + connection buffer: 90/stop · 180/2-stop), within rounding
  • Seasonal dot border colour matches the nonstop colour (seasonal is the nonstop variant)

💰 CURRENCY GUIDE

validate_currency.py audits Travel-Website/Trip-Essentials/Currency-Guide.html (generated by Brain/scripts/build_currency.py — static country→currency map in-script + live USD rates). Never hand-edit the page; change the script and rebuild. Run: python3 Brain/scripts/build_currency.py then python3 Brain/scripts/validate_currency.py (exit 0 = clean). Full format spec: Trip-Essentials/Essentials-Pages-Rules.md § Currency Guide.

  • Country coverage — the country set equals the guides-index flag set exactly (no more / no less); every guide city maps to one block; no city under two countries
  • "Last update {Mon Year}" stat pill present and not older than the current month (the monthly refresh keeps it current)
  • Index table = four columns Country · Currency · Sym · US$1; colgroup 28/40/12/20; header row uses the body font size (not the small label size)
  • Sym and US$1 columns right-aligned; numbers are bare (no symbol) + tabular; index symbols use body colour (no accent override)
  • Block header is one line — name | currency | code | symbol — with gold #8a6c1a pipe separators and all other text a single brown #5C3D11; Guides sits on the same line
  • A converter on every non-USD block; its output matches the US$ label (no bold / no own colour); the standalone "US$1 ≈ …" rate line stays removed
  • Typography uses the --fs-* scale (no hardcoded pixel font sizes); no off-palette green colours
