§ 9. Font sizes — mobile

Applied at max-width: 600px. Mobile font sizes match desktop — no scale-up (locked 2026-06-03). The mobile block adjusts padding and layout only; every element inherits the same 14px base, so pills, titles, and boxes are identical in size to desktop.

§ 15. Index stat row — Guides-Index.html (2026-06-14)

Directly under the Travel Guides banner sits one stat row holding two small grey text lines — place counts on the left, country counts on the right. They are computed by JavaScript from the cards (never hardcoded), so the place numbers always reconcile: total guides = visited + on the list.

ElementStandard
Row container.stat-row — display:flex; justify-content:space-between; align-items:baseline; gap:16px; margin:-6px 0 24px (sits close under the banner, with space below before the search bar)
Both lines.stat-line — font-size:12px; color:var(--muted); margin:0; numbers bold via .stat-line b { color:var(--text); font-weight:600; font-variant-numeric:tabular-nums }
Left line (places)#guide-count — <b>N</b> guides | <b>N</b> visited | <b>N</b> on the list (places = guides; visited = been, on the list = want)
Right line (countries)#country-count with .stat-line--right (text-align:right) — <b>N</b> countries | <b>N</b> visited
Separatora vertical bar | between each number

Important: place data and country data stay on separate lines — never merged into one set of numbers. No pills, dots, or buttons; both lines are plain small grey text. Locked 2026-06-14 (by request) — enforced by brain_check.py check_index_stat_row.

§ 16. Title banner standard — all non-guide pages (2026-06-15)

Every Trip Essentials page, the Guides index, and the Main Pages hub carry the same title banner: the warm terracotta gradient band (#7a3b1e → #b85c2a → #d4874a) with the page title as a single <h1> — 14px, bold, uppercase, white, letter-spacing 0.04em (the shared --fs-title in assets/_travel_style.css). All title banners must match — no page may size its title differently.

  • One size everywhere — 14px. A page may keep an inlined copy of the banner rule (some pages don't link _travel_style.css), but the banner <h1> font-size must be 14px / var(--fs-title). A banner enlarged only inside @media print is allowed.
  • Title only. No emoji and no subtitle inside the banner band — the <h1> text alone (eyebrow/label elements are hidden site-wide).
  • Universal toolbar. The shared nav is the one assets/toolbar.js on every page. Guide-page CSS stays separate (guide_v3.css) and is out of scope for this banner rule.

Enforced: brain_check.py → check_banner_title_size hard-fails any non-guide shareable page whose title-banner <h1> font-size isn't 14px.
