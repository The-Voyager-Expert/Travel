🗂️ Guide Structure

Phase: 2 — Guide structure

Summary: The guide structure — read top to bottom as you build. Each section of the guide points at the rule file that owns its rules.

§ 1. Build phases & required reads

A guide build moves through phases. Each phase has a mandatory read list. Skipping a phase's reads is how guides drift.

Phase 0 — Session start

Before anything else, every session.

  • Rules for Claude.html

Preflight gate (after Phase 1–2 reads, before first HTML line): Run python3 Brain/scripts/guide_tools.py preflight <City>. This verifies all required Phase 0–2 reads are checked off in _build/build_state.md before any guide HTML is written. Non-zero exit = incomplete reads — complete them, re-run, proceed only on exit 0. Writing guide HTML before exit 0 is a violation.

Build-state tracker (created at the start of every build)

The first action of any guide build is to create a build-state tracker at Guides/{City}/_build/build_state.md. Claude updates it as the build progresses; the validator (and Ship Checklist.html) reads it to confirm every phase actually fired. Format:

# Build state — [City]
Started: [ISO Date]
Last updated: [ISO Date]

## Phase 0 — Session start
- [ ] Rules for Claude.html

## Phase 1 — Technical prerequisites

- [ ] Photos Rules.html
- [ ] Connectors.html (Brain/Reference/)

- [ ] Brain/Reference/Ship Checklist.html

- [ ] every extra populated or carries negative-finding line (conditional sections — Heads Up, Claude Inspiration, Skip List — exempt when empty)

Each [ ] flips to [x] when the file has been read or the step completed. The tracker is plain markdown — easy to scan, easy for the validator to parse, easy for Claude to update inline.

Phase 3 — Day shape

Before locking any day.

  • Day Structure.html

Phase 4 — Per-stop build

Every stop in the itinerary.

  • Tickets.html — for any ticketed stop (🎟)
  • Motion Rule.html
  • Icon Order and Format.html

Phase 5 — Per-section build

For each extra section that ships.

  • The matching extra-section rule file (see below for the section list)
  • Motion Rule.html — when walk/ride times appear
  • Icon Order and Format.html
  • Brain/Reference/Story-Pages.html — the mandatory Read About companion page; build it in the same pass as the extra sections, never deferred

Patch-build mode

A patch session is a session that edits existing guide HTML but makes no structural changes — copy corrections, URL swaps, minor text fixes. Patch sessions use an abbreviated read set:

  • Required: Phase 0 (Rules for Claude.html) + Phase 6 (ship gate below).
  • Skipped: Phase 1–5 reads.

A session that adds a new stop, adds or removes an extra section, changes the day structure, or edits the HTML template in any structural way is not a patch session — run the full Phase 1–5 reads for those changes.

Phase 6 — Ship gate

Before the guide outputs — and before any "done" claim in chat.

  • Run Brain/Reference/Ship Checklist.html end-to-end. Every "no" blocks ship.
  • Validator passes (Brain/scripts/validate_itinerary.py)
  • Every extra section is populated or carries its negative-finding line — except the conditional sections (❗️ Heads Up, Claude Inspiration, Skip List), which ship nothing when empty
  • Run the 5-step ship tail: python3 Brain/scripts/guide_tools.py update-index <City> — prints a step-by-step checklist (Guides-Index.html card, prev/next wiring, counts, toolbar data, map pin) and verifies each step. All 5 must be done before the guide is shipped.

Patch sessions and crash recovery: Phase 6 applies to any session that modifies guide HTML, not only full builds. When picking up a guide mid-build, read _build/build_state.md first — an unchecked Phase 6 means the guide is not done. Run the full ship gate before declaring it complete.

§ 2. Structure

The guide ships these sections in order. Each section's rules live in its rule file.

  1. Title Page — City + Hotel + Address. See Hotel Banner.html.
  2. Trip Overview — clickable day cards, with the extras pill row at the end pointing to each extra section's anchor. See Trip Overview.html.
  3. Day blocks — Day 1 → Day 2 → … → Day N. Built from these rules:
    • Day shape (opener, closer, transitions). See Day Structure.html.
    • Stop types, day-overview labels, stop block format, and selection criteria. See Stops Structure.html.
    • 🎟 Ticket boxes. See Tickets.html.
    • Transit banners (motion banners across the guide — 🚶 walk, 🚕 ride, 🚎 tram, 🚝 metro, day openers and closers). See Motion Rule.html.
    • Icon order and per-icon format. See Icon Order and Format.html.
    • Stop photos. See Photos Rules.html.
    • URL verification and address format. See Links.html.
  4. Extra sections, in order:
    1. 🗓️ Weekly Closures (city-wide closures by weekday). See Weekly Closures - Extra Section.html.
    2. 📅 Tours (guided tours in the destination). See Tours - Extra Section.html.
    3. ☕ Cappuccino (best espresso near the hotel). See Cappuccino - Extra Section.html.
    4. 🫕 Restaurants Near Hotel (closest restaurants worth a meal). See Restaurants Near Hotel - Extra Section.html.
    5. 🍽️ Downtown (best restaurants in the historic core). See Downtown Restaurants - Extra Section.html.
    6. 🍮 Local Tastes (foods and drinks famous in the destination). See Local Tastes - Extra Section.html.
    7. 🚗 Food Delivery (delivery apps available in the destination). See Food Delivery - Extra Section.html.
    8. 🎭 Shows, Performances & Concerts (destination-level performances). See Shows, Performances & Concerts - Extra Section.html.
    9. 🚌 Getting Around (in-city transit — ride apps, tram, metro). See Getting Around - Extra Section.html.
    10. 🚆 Train Stations Near Hotel (rail stations within reach). See Train Stations Near Hotel - Extra Section.html.
    11. ⛲️ Day Trips by Train (train trip destinations from the guide city). See Day Trips by Train - Extra Section.html.
    12. 🏓 Pickleball (courts within walking distance) — California, Arizona, and Oregon only. See Pickleball - Extra Section.html.
    13. ⭐ Michelin Restaurants (star-rated dining). See Michelin Restaurants - Extra Section.html.
    14. ❗️ Heads Up (venue surprises and timing traps) — conditional on entries. See Heads Up - Extra Section.html.
    15. Claude Inspiration (free-form observations from the trip — icon, title, and color theme are Claude's pick per guide) — optional. See Claude Inspiration - Extra Section.html.
    16. Skip List (footnote naming venues skipped because already visited — small italic grey, no banner, no heading; ships only when the destination has a skip list). See Skip List - Extra Section.html.
  5. Also on this site — the closing pill row at the very bottom of every guide, after all extra sections and the Skip List. Ships on every guide, no exceptions.
    • Header: "Also on this site" — exact text, never changed.
    • Each pill is an also-on-this-site-pill link (the CSS-styled, gold-bordered class in guide-style.css; the retired trip-resource-pill is unstyled and forbidden). Icon AND section name are both locked — neither may change independently.
    • Canonical pill set (icon · name — both must be exact):
      • 🌤️ Weather
      • 🕐 Time Zones
      • 🔌 Plug Adapter
      • 💰 Currency
      • 🛡️ Safety Guide
      • 🪪 Visas
      • 📊 Stats Across Europe / 📊 Stats Across US / 📊 Stats Across Asia / 📊 Stats Across Canada / 📊 Stats Across the Caribbean / 📊 Stats Across South America — region-appropriate, one pill only (label = the live toolbar form; "Caribbean Stats" / "Stats · South America" are wrong)
      • 🚆 European Train Guide — European guides only; always last
    • All seven base pills are required on every guide — Weather · Time Zones · Plug Adapter · Currency · Safety Guide · Visas · Stats. Absence of any one is a failure, not just wrong order. 🚆 European Train Guide is the only region-gated pill (EU only).
    • No suffixes, no rewording, no added country or type. "💰 Currency · Switzerland", "🔌 Plug Adapter · G", "📊 Stats Across the US" — all wrong. The name is always the base label, full stop.
    • Enforced by brain_check.py check_also_on_site_pill_labels (every session start — hard-fails on icon change, name change, unknown/banned pill), validate_itinerary.py TR-2/TR-4/TR-6 (ship gate — pill class, order, and the full required set), and Brain/scripts/validate_also_on_site_pills.py (standalone backstop; exit 0 = all shipped guides clean).
