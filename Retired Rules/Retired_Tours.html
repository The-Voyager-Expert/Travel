📅 Tours — Retired Stop Type

⚠️ RETIRED (2026-05-20): Tours as an in-stop type are no longer built in new guides. The 📅 tour box and Guided Tour Stop pattern are retired. Skip-the-line tickets (🎟) remain active. This file is preserved for reference.

Phase: 3 — Per-stop build (retired)

Summary: How to find, qualify, and format guided tours at every stop. Retired — tours are now a guide-level Extra Section only (see Tours - Extra Section.html).

§ 1. Preference order

At every stop — self-guided is the fallback, not the default. Always search for a qualifying tour before defaulting to anything else; even lesser-known stops often have one.

  1. Tour — someone guides (see for format).
  2. Skip-the-line ticket — self-walk, skip the line (see Tickets.html).
  3. Walk-up — only when nothing else qualifies.

§ 2. Sources

  • First tier: Viator · GetYourGuide · TripAdvisor (flat pool, no order).
  • Fallback: any popular local operator (Asia: Klook).

Viator — MCP connector

  1. search_experiences — natural-language searchTerm, trip start/end dates, currency EUR, locale en-US. Returns up to 10 results with rating, reviewCount, fromPrice, duration, freeCancellation flag, and click-out URL.
  2. get_experience_details — call with the code from step 1 when full details on a specific experience are needed.

Viator MCP tools are deferred — load schema with ToolSearch before calling. Use a stable sessionId per conversation (e.g. [city]-[year]) and reuse across both calls.

GetYourGuide and TripAdvisor — browse directly

No connector available for tour searching. Use Claude in Chrome MCP to browse search results and product pages directly.

§ 3. Rating bar

4.5+⭐ · minimum 6 reviews — everywhere. Rating must be visible at page level; not visible = treated as 0.

When rating is not visible:

  1. Try a different WebSearch query (e.g., site:viator.com [product-code] rating reviews).
  2. Escalate to Chrome MCP — live page render.
  3. Last resort — confirm on the live page manually.

§ 4. Tour boxes per stop

Target 3 tour boxes per stop, ranked by rating + review count. Ship fewer only when 3 qualifying tours can't be found — validator warns.

§ 5. Tour stop types

The address should provide the street name and neighborhood. The city is included only when the address is in a city different from the destination [City]. No postal code or country.

§ 6. Hotel Pick Up / Drop-Off (🚐)

When the tour includes hotel pickup or drop-off, add the matching line to the tour box, immediately after the tour title.

🏨 ↔ 🚐   round-trip (hotel pickup + drop-off)
🏨 → 🚐   pickup only (hotel → tour start)
🏨 ← 🚐   drop-off only (tour end → hotel)

§ 7. What follows a guided stop

Case 1 — Operator brings you back (🏨 ↔ 🚐 or 🏨 ← 🚐)

No motion banner needed after the stop. The tour ends at the hotel.

Case 2 — Operator does not bring you back, and this is the last stop of the day (🏨 → 🚐 or no pickup line)

The tour ends at the venue, not the hotel. Add a motion banner back to the hotel.

🚶 N min · 🚕 M min → hotel

Case 3 — Guided stop is not the last stop of the day

Another stop follows. Add a between-stops motion banner to the next stop.

🚶 N min · 🚕 M min → [Next Stop]
