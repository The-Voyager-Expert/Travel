Finding — Validator Index Documentation Error

FIXED   Validator Index line 894 — Claude Inspiration theme-{color} rule

What was wrong

The Validator Index documented the Claude Inspiration theme-{color} class as REQUIRED — stating the outer div must include a theme-{color} modifier. This was the opposite of what both CORE RULES and the validator code actually enforce.

Root cause

The 2026-07-17 Part 17/20 audit reversed the polarity of the T_NEW5 check from "MUST NOT have theme-{color}" to "MUST have theme-{color}." The owner confirmed same day that the reversal was incorrect (CORE RULES always required plain white background, no modifier). The code was reverted correctly, but the Validator Index was only updated for the reversal — not for the subsequent revert. Line 894 remained stuck in the "MUST have" state.

CORE RULES authority

  • Claude Inspiration - Extra Section.html § 2: "The section renders on a plain white background. No theme-{color} class is applied to the outer div — the correct format is class="claude-inspiration" with no modifier."
  • Guide Structure.html § 2 item 15: "always renders on a plain white background, no theme-{color} class"
  • validate_itinerary.py T_NEW5 (lines 24486–24507): correctly hard-fails when any theme-{color} is present on the Claude Inspiration outer div

Fix applied

Line 894 in Brain/Reference/Validator Index.html updated to correctly document:

  • The theme-{color} class is FORBIDDEN on the Claude Inspiration outer div
  • Correct format is class="claude-inspiration" with no modifier
  • T_NEW5 hard-fails when any theme-{color} is present
  • Full history of the 2026-07-12 / 2026-07-17 / 2026-07-18 changes documented inline
