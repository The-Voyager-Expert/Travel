Observations (non-blocking)

22 shipped guides have stray mobile.css link in head. CLAUDE.md states "LOCKED 2026-06-12: new guides link guide-style.css only, not mobile.css" — but this is a behavioral rule, not a CORE RULES structural rule. The validator does NOT check for this because: (a) mobile.css is idempotent/harmless in a guide head (guide-style.css already covers it); (b) fixing 22 guides is fleet-scope work outside this audit's remit. Noted for a future cleanup pass.

Affected guides include: Amsterdam, Bend, Bergen, Berlin, Bhutan, Bologna, Bora-Bora, Bordeaux, Boston, Boulder, Bruges, Brussels, Budapest, Buenos-Aires, Cairo, Cambridge, Cancun, Cannes, Cape-Cod and others (22 total).
CI skip mode + CORE RULES content checks: minor documentation gap. The Validator Index section note says "All CORE RULES integrity + content checks are skipped in published-only/CI mode." In code, the content checks (banned phrases, drift words) are gated on if _cr_integrity_ok and _cr_rules_dir.is_dir() — meaning they WOULD run in skip mode if the CORE RULES dir were mounted. In practice the CORE RULES dir doesn't exist in CI, so the behavior is identical. This is a documentation imprecision, not a functional gap. A future DriftyCat correction to the Validator Index note would clarify this.
