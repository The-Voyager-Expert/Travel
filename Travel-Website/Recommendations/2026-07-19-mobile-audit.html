ANALYSIS
Root cause of the iOS hamburger rendering issue: iOS Safari browser cache
Auditing all toolbar.js commits back through the past week: the text: '📊 Stats Across the Caribbean' entry has been correct in every commit — the source code was never wrong. The two occurrences of the missing label were therefore a browser-side cache issue: iOS Safari served a stale, pre-cache-bust version of toolbar.js. The ?v=101 cache-bust in commit cc46d4cf2 (2026-07-19) forces a fresh load. The new Layer 8 and brain_check gate still have value: they catch the class of source-code regression where a future crib accidentally sets an emoji-only text: value — a bug the browser-cache issue had masked as a possibility.
