These are suggestions for a future dedicated pass — not in scope for a content audit run.
🔼 Sticky search + filter controls
Look / UX
The search input and filter pills scroll off-screen on long card lists. Adding position: sticky; top: 48px (below the toolbar) with a var(--bg) background fill and a subtle bottom shadow on the .controls div would keep the filter bar accessible at all times. This is purely CSS — no structure changes needed.
🔼 Bilt card — two-tier mini-table format
Content clarity
The Bilt 2.0 entry now describes two tiers (Blue $0 and Obsidian $95) in a single prose note. The page already uses .mini-table for multi-card comparisons (e.g. the Hilton section). Converting the Bilt note to a two-row mini-table (columns: Tier · Annual fee · Earn rate · Best for) would match the page's own pattern and be much easier to scan.
🔼 Add transfer-partner count to Amex Platinum and Gold notes
Content — missing key differentiator
The Amex Platinum and Gold notes list benefits but don't mention the 22 airline and hotel transfer partner ecosystem — a primary reason these cards are chosen for travel. Chase notes often call out "14 transfer partners." Adding a concise "22 airline and hotel transfer partners" phrase to each Amex note would close a noticeable information gap.
🔼 Card-level visual tier indicator
Look / scannability
All card rows currently look identical. A 3px left border accent (e.g. gold for premium annual-fee cards, muted grey for no-fee) on .card-row would create at-a-glance tier identification without restructuring anything. Alternatively, a "Premium" / "No fee" badge-top / badge-ok label next to the card name. Purely additive CSS + one badge class per row.
🔼 Autograph / Autograph Journey cross-reference note
Content — user navigation
The Wells Fargo Autograph ($0) and Autograph Journey ($95) share an identical transfer partner ecosystem but different earn categories. A short cross-reference sentence on each entry ("Same transfer partners as the Autograph Journey" / "Same transfer partners as the Autograph, $0 version") would help readers evaluating which tier is right for them without re-reading both notes.
