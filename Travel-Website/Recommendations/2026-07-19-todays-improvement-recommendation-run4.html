check_css_duplication.py scans Recommendations/ audit reports, flooding brain_check with 193 false-positive WARN hits every session

What's happening

brain_check.py fires a persistent ⚠ [css-duplication] 193 duplicated CSS selectors WARN at every session start. The culprit is check_css_duplication.py, which scans all 825 HTML pages under Travel-Website/ — including every file in Travel-Website/Recommendations/.

Each audit report defines its own local utility CSS: .ok, .pass, .warn, .stat-num, .stat, .green, .fixed, .clean, .report-header, .report-meta, .fail, .num, .info, .report-title, .finding, .stat-card, and more. Because these class names repeat across dozens of report files, the script treats each one as a violation of the SHARED CSS ONLY rule.

These are internal owner-facing documents — not site components. They are not loaded by any live page. Flagging their local utility CSS as "component duplication" is a scope error, not a real violation.

Why it matters

The WARN has become invisible noise. The count has grown from 93 (noted as "persistent background issue" in the 2026-07-17 mobile audit) to 193 today — and will keep climbing as new reports are generated. Because 193 phantom hits fill the output, any real CSS duplication in live site pages (Cards-ATM.html, Trips.html, Weather.html, Visas.html, Safety-Guide.html, Passport.html, Travel-Packing.html, Destination-Records.html) is buried and effectively invisible.

The SHARED CSS ONLY rule exists to catch drift in the live site. Audit reports being flagged defeats that purpose entirely.

The fix

Add a path-level exclusion for Travel-Website/Recommendations/ in the page-scanning loop of Brain/scripts/check_css_duplication.py. The script already has DEFERRED_SELECTORS (class-name allowlist) and GENERIC_SELECTORS (tag allowlist), but no folder-level exclusion.

Near the top of the script, add an excluded-paths set alongside the existing constants:

EXCLUDED_DIRS = {
    'Recommendations',
}

Then in the page-discovery loop, skip any page whose path passes through an excluded directory:

# Before the existing per-page processing:
if any(part in EXCLUDED_DIRS for part in page_path.parts):
    continue

This keeps Recommendations/ entirely out of the scan. No existing exclusion mechanism needs touching — the DEFERRED_SELECTORS list can even be trimmed if the report-specific classes were added there as a workaround.

After applying: run python3 Brain/scripts/brain_check.py and confirm the css-duplication WARN is gone. If any real site-page duplication exists, it will surface cleanly on this same run.

If desired, also consider whether any Recommendations-specific class names currently in DEFERRED_SELECTORS were added as a partial workaround — those entries can be removed once the folder is excluded.
