OBSERVATION 3 items in LINKS section are logically STOPS / ICONS checks

Items 15 (no-market stop), 16 (⏰ avg time), and 17 (🎒 retired) appear in the LINKS section of the Validator Index but their source rules are in Stops Structure.html and Icon Order and Format.html, not Links.html. These are correctly implemented — they just appear under the wrong heading in the index. This is a minor documentation classification issue, not a functional gap. Moving them to their correct sections would improve index readability but requires no validator code change.

FIXED update_validator_coverage_artifact.py — list vs dict type error

The script assumed active_claims in the state file was a dict {part: {crib, timestamp}}, but the current state file stores it as a plain list [part, ...]. The script threw AttributeError: 'list' object has no attribute 'items' when attempting to regenerate the artifact. Fixed: added a type check — when active_claims is a list, all items are treated as live claims (no timestamp filtering). The fix is backward-compatible with both formats.
