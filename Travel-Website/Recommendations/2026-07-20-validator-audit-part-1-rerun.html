Finding — Fixed

Fixed
Phase 5 build_state check missed claude-inspiration section — spec file never required in tracker

The Phase 5 _bs_shipped_ids scan detected extra sections by looking for class="extras-section" div tags only. The Claude Inspiration section uses class="claude-inspiration" (never class="extras-section"), so it was silently excluded from the scan. As a result, Claude Inspiration - Extra Section.html was never added to _bs_p5_required, and the build_state check never enforced that the spec was read — even though the section is mandatory on every guide since 2026-07-12.

Root location: Brain/scripts/validate_itinerary.py lines 1031–1035 (Phase 5 shipped-ids detection loop)

            # Phase 5 — required per shipping .extras-section id.
            _bs_shipped_ids: set[str] = set()
            for _tag in re.findall(r'<div\b[^>]*>', html, re.IGNORECASE):
-               if re.search(r'\bextras-section\b', _tag, re.IGNORECASE):
+               if (re.search(r'\bextras-section\b', _tag, re.IGNORECASE)
+                       or re.search(r'\bclaude-inspiration\b', _tag, re.IGNORECASE)):
                    _id_m = re.search(r'\bid\s*=\s*"([^"]+)"', _tag, re.IGNORECASE)
                    if _id_m:
                        _bs_shipped_ids.add(_id_m.group(1))

Since claude-inspiration is already keyed in BUILD_STATE_PHASE_5_MAP at line 909, no further code changes were needed — adding it to the scan set automatically wires it through to _bs_p5_required and the _bs_phase_check call.
