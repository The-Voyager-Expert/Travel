Site Drift Log

Infrastructure & content routines · Recurring patterns only · Last updated: 2026-07-23

How to use this file

This file is for recurring patterns — not one-time findings. If you ran into something once, log it to To Do List/To_Do_List.md instead.

Log here when a routine sees the same problem across multiple runs. That's drift: something that keeps coming back despite not being fixed. The pattern itself is the signal.

To add an entry: append a .log-row inside the matching .entry-log, or create a new .entry block if this pattern hasn't been logged before. Use the status badges: Watching (seen but not escalated), Escalated (logged to To Do List for action), Resolved (root cause fixed).

Toolbar version lag on auto-generated and static pages Escalated
Auto-generated pages (Travel-Stats.html, Safety-Guide.html) embed a stale toolbar.js?v=NNN version string baked into their build scripts. Every time toolbar.js bumps version, these pages fall behind and brain_check warns on them. Static pages similarly accumulate version lag between refreshes. The root cause is that the toolbar version string is duplicated in multiple build-script templates (build_travel_stats.py, build_safety_guide.py) rather than read from a single source. Seen across scripts-docs-audit, rule-compliance-audit, claudemd-drift-audit, validator-audit, and brainmd-audit.
