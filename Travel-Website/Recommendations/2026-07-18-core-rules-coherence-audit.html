Owner Item — C-21 NEW: Claude Inspiration § 2 internal contradiction (color theme)

Claude Inspiration - Extra Section.html § 2 heading says "§ 2. Icon, title, and color theme" and the opening sentence says "Claude picks the icon, section title, and color theme per guide."

Yet later in the same § 2: "The section renders on a plain white background. No theme-{color} class is applied to the outer div — the correct format is class="claude-inspiration" with no modifier."

The validator correctly enforces no theme class (hard-fail since 2026-07-12). The CORE RULES file has an internal contradiction in § 2. Proposed fix: rename § 2 heading to "§ 2. Icon and title" and change the first sentence to "Claude picks the icon and section title per guide."

Owner Item — C-22 NEW: Trip Overview § 3 Heads Up pill uses bare ❗ (missing U+FE0F)

Trip Overview.html § 3 table shows the Heads Up pill as ❗ Heads Up (bare ❗, U+2757 only — no variation selector U+FE0F).

Both Heads Up - Extra Section.html and Icon Order and Format.html use ❗️ (U+2757 + U+FE0F). The validator enforces ❗️ with VS16 everywhere via a global bare-❗ ban (hard-fail since 2026-06-05). The validator's canonical label dict also shows "heads-up": "❗️ Heads Up" with VS16. Trip Overview is the one file inconsistent with every other source. Proposed fix: update Trip Overview.html § 3 table Heads Up cell to ❗️ Heads Up.

Owner Item — C-23 NEW: Icon Order § 3 shows Workaround row — contradicts Heads Up file (row is BANNED)

Icon Order and Format.html § 3 includes a Workaround: [What To Do Instead] row as part of the Heads Up entry format.

Heads Up - Extra Section.html § 2 shows NO Workaround row — only [Venue] — [Short Title] and ↳ [Note] ≤150 chars.

The validator explicitly bans the Workaround row (cg_workaround_banned check — hard-fail if present). The Workaround row was removed from the format at some point and cleaned from the Heads Up file and validator, but the stale reference was not removed from Icon Order. Proposed fix: remove the Workaround row from Icon Order and Format.html § 3 Heads Up format block.

Owner Item — C-18 (carried over): Rules for Claude.html predates the 2026-07-12 delete policy

Rules for Claude.html §2 DriftyCat says: "🚫 Deletion loop — never rm a Drive-only file. Untracked files cannot be recovered from git. Move them to Travel/archive/ — never rm."

CLAUDE.md § 3 (updated 2026-07-12) now has a two-track rule: git-tracked files → delete freely (git history is the archive); Drive-only files → move to Travel/archive/. Rules for Claude.html predates this split and still frames everything as "archive-only." Proposed fix: update Rules for Claude.html §2 to add the two-track distinction and align with the 2026-07-12 CLAUDE.md policy.
