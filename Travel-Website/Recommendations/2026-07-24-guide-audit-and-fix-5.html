Validator Changes

TR-8b — WEATHER FRAGMENT CASE-SENSITIVE
Updated in validate_itinerary.py · hard-fail · 2026-07-24

What changed: TR-8b previously compared Weather.html #fragment values case-insensitively (both fragment and CLIMATE_DATA key were lowercased). But Weather.html resolves anchors via JS cityKeys.indexOf(hashKey) — which is case-sensitive. So #zakynthos silently failed to scroll to the 'Zakynthos' CLIMATE_DATA entry at runtime, even though the validator passed it.

Fix: _tr8b_wx_keys now stores original-case keys; the Weather fragment comparison is exact-match. When a fragment matches case-insensitively but not exactly, the error message suggests the correct-case form. Safety-Guide comparison is unchanged (uses lowercase data-city values — case-insensitive is correct there).

Trigger: Zakynthos shipped 2026-07-23 with #zakynthos while CLIMATE_DATA key is 'Zakynthos'. Only 1 of 226 guides was affected. Fix confirmed: Zakynthos now fails on the wrong case; Crete (correct case) still passes.

ALT TEXT — ST./MT./DR. ABBREVIATION FALSE-POSITIVE FIXED
Updated in validate_itinerary.py · warn-tier · 2026-07-24

What changed: The Q-warn alt text sentence detector used re.search(r'[.!?]', alt_val) on the raw alt string. This correctly flags real descriptive sentences but also false-positives on standard title/geographic abbreviations: "St. Nikolai Memorial", "Mt. Fuji", "Dr. King Memorial" all contain a period but are valid subject-name alt texts per Photos Rules.html §8.

Fix: Common abbreviations (St|Mt|Dr|Ft|Mr|Mrs|Ms|Jr|Sr) followed by a period and whitespace/end-of-string are stripped before the period check. Hamburg's "St. Nikolai Memorial" and "St. Michaelis Church" were the triggers — both are now correctly assessed as clean. Real sentence-like alt texts still fire the warning.
