Bug Fixed

FIXED Dead overview plus-connector check — bad_plus_in_overview never fired

File: Brain/scripts/validate_itinerary.py lines 4451–4458

Impact: The no-+ connector check was documented as covering both .stop-name spans AND .overview-day-title overview cards. The overview half was dead code — a regex requiring the 🎒 backpack emoji in the title text, but 🎒 has been banned from all guide HTML since 2026-06-16. The overview check silently passed every guide for over a month.

Fix: Replaced the 🎒-gated regex with a scan of all .overview-day-title divs. Extracts the stop-list portion (everything after the en-dash – separator), skips Train Day cards (🚆 in stop portion), and hard-fails on \w\s*\+\s*\w in the stop list.

Validator Index: Two references to the retired .overview-day-stops element name corrected to .overview-day-title (the element was renamed 2026-05-26; the Index retained the old name).

Fleet impact: Preventive — 0 current guides use + connector in overview cards. The check now actually fires and enforces going forward.
