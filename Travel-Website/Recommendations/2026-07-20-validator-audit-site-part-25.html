Half 2 — Validator Bug Fixes (validate_regional_stats_chips.py)

VALIDATOR Bug 1 Fixed — Duplicate-section overwrite → merge

Root cause: sections[title] = countries — Europe-Stats repeats section titles ("Population", "Happiness Index", etc.) for three sub-grids: European countries, Oceania (Australia/NZ), and Africa (Egypt/Morocco/Seychelles). The last occurrence (Africa) overwrote the European entry, making the merged section only contain {egypt, morocco, seychelles}. This caused Czech Republic and all main European countries to appear absent from every repeated section (17 false positives per European guide).

Fix: Changed sections[title] = countries to sections.setdefault(title, set()); sections[title] |= countries. All duplicate-titled sections now merge their bar countries, giving one complete set.

Impact: Eliminated false positives for Czech Republic across 17 sections, plus similar false positives for any guide from countries covered in the primary European sub-grid.

VALIDATOR Bug 2 Fixed — Partial chip key mismatch

Root cause: _match_country returns the chip key (e.g. "czech" from data-q="czech") but bar section countries store full names (e.g. "czech republic"). The present check used Python set membership ("czech" in {"czech republic", ...} → False), so Czech Republic was always found in zero sections, and ALL sections were reported as missing.

Fix: Added _country_in_set(matched, countries) helper that performs substring matching (same logic as _match_country) so partial chip keys correctly identify full bar country names.

VALIDATOR Bug 3 Fixed — Regional subsections incorrectly included in peer-overlap

Root cause: Sections with "Region · Base Title" format (e.g. "Mexico & Central America · Stayover Air Arrivals (per year)") were included in the peer-overlap check. After Mexico was added to Caribbean-Stats, Aruba and Barbados were flagged as missing from this Mexico-specific subsection because enough of their peers (Mexico + Costa Rica) were in it.

Fix: Added a guard: skip any section whose title contains " · " in the missing-section check. Regional subsections are geographic subsets by definition and should never flag countries from other regions.

Secondary fix: The _covered_by_subsection() helper (added earlier) handles the inverse — countries that ARE in a subsection are correctly considered present for the base section requirement.

VALIDATOR KNOWN_EXCLUSIONS mechanism added

Added a KNOWN_EXCLUSIONS: dict[tuple[str, str], set[str]] dict to suppress intentional data gaps where a country is correctly absent from a section because the source data does not include it. Current entries: Monaco excluded from Europe-Stats Happiness Index and Europe-Stats Tertiary Education Attainment. Future intentional exclusions can be added here without page data fabrication.
