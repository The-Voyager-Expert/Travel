UX Handoff — Delta Routes — 2026-07-16

Source: delta-routes-audit · Date: 2026-07-16 · Pages: Delta-Routes-SEA.html · Delta-Routes-Full.html
1 · `:root {}` defined in inline page CSS on both pages
File: Delta-Routes-SEA.html AND Delta-Routes-Full.html
Element: <style> block at top of both pages, lines 11–14 (SEA) / 12–15 (Full)
Issue: Both pages define a :root {} block (tier routing color variables --c0, --c0bg, --c1 etc.) in their inline <style>. Site rule: "NEVER define body {}, * {}, :root {} in pages — these belong ONLY in web-travel-style.css."
Fix: Move all six custom properties (--c0/--c0bg/--c0bd, --c1/--c1bg/--c1bd, --c2/--c2bg/--c2bd) into Travel-Website/assets/web-travel-style.css, since these variables are used identically on both Delta pages (and could be reused elsewhere). Replace the in-page :root {} block with a comment pointing to the shared file.
2 · `#fdf3e0` (banned beige) used as tier-2 card/pill background on both pages
File: Delta-Routes-SEA.html AND Delta-Routes-Full.html
Element: --c2bg: #fdf3e0 in the :root block, applied via .tier-pill.t2, .tier-badge.t2, .dest-card.t2
Issue: #fdf3e0 is explicitly listed in DriftyCat as a banned beige background: "A pill/button REST state is WHITE, never beige — #fdf3e0 (and siblings) are pure drift." The tier-2 amber family currently uses this color as the background fill for its pills, badges, and card accents.
Fix: Replace --c2bg: #fdf3e0 with --c2bg: #fff8ee (a warm white-leaning amber that reads as "amber tier" without being the banned beige). Or better — make --c2bg a transparent tint: rgba(200,164,74,0.10) which achieves the same yellow-amber coding against a white card background, with no beige drift. Check visual rendering after change.
3 · Hardcoded hex colors in inline CSS on both pages
File: Delta-Routes-SEA.html AND Delta-Routes-Full.html
Element: <style> block throughout — .nav-link, .filter-input::placeholder, .tag.d1, .hub-pill, and others
Issue: Both pages hardcode hex values in property declarations: #A8895A (placeholder color), #c8b99a (focus border), #f8e8e8/#7a1010 (d1 tag), #f8ece8/#e8c0b0 (hub-pill background/border), #3d3a32 (nav-link color), etc. Site rule: "NEVER hardcode hex colors in pages — use CSS variables."
Fix: Replace with the matching CSS variables from web-travel-style.css: #3d3a32 → var(--text), #e6e2da → var(--border), placeholder gold → var(--accent). Tag colors (#f8e8e8, #7a1010) don't have existing vars — add --c-warn-bg/--c-warn to the shared file and reference them. Hub-pill can use var(--warm) + var(--border2).
4 · Active tier-pill state uses only dark shadow, not terracotta active state
File: Delta-Routes-SEA.html AND Delta-Routes-Full.html
Element: .tier-pill.active { font-weight:700; box-shadow: 0 0 0 2px rgba(0,0,0,.30); }
Issue: When a tier pill is selected, it gains bold + dark ring but no background change. The site rule requires active/selected interactive states to use terracotta #b85c2a. These tier pills are clickable filters (line 79 SEA: "clickable tier pills") — they are interactive state pills, not just a passive legend. The active state should reflect the site's terracotta active pattern.
Fix: Update .tier-pill.active to: background: linear-gradient(135deg,#7a3b1e 0%,#b85c2a 55%,#d4874a 100%); color: #fff; border-color: #b85c2a; — matching the same .nav-link.active style already on both pages. Drop the dark box-shadow.
5 · "Updated July 2026" stamp floats outside the header block on SEA page
File: Delta-Routes-SEA.html only
Element: <span class="updated-stamp">Updated July 2026</span> at line 132, inside a standalone .wrap div after the header's closing </div>
Issue: The updated stamp sits in its own separate wrapper after the header, rather than inside the header block. On the Full page, the stamp appears inline in the header correctly. This creates inconsistent placement between the two pages.
Fix: Move the <span class="updated-stamp"> to inside the .header div on the SEA page, matching the Full page's layout. The stamp's injection is normally handled by toolbar.js via data-updated — check whether the SEA page has a data-updated attribute on the toolbar-mount div; if not, add it instead of a manual stamp.
