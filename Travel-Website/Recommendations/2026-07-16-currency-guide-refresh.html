Improvement ideas

Region jump navigation Function

What The page has six region headers (Europe, Asia, Americas, etc.) acting as visual dividers, but there are no anchor links to jump between them. On mobile — and on desktop with 71 entries — a user looking for Japan has to scroll past all of Europe and the Americas to get there. The region headers already have clear labels and the country blocks already have scroll-margin-top: 72px set, so the infrastructure is in place.

How Add a compact row of region pills just below the search input — one per region (Americas · Europe · Middle East · Asia · Pacific · Africa) — each linking to an anchor on the matching .region-section. Uses the existing pill class and site token colors; no JS required.

Currency symbol search support Function

What The search input placeholder says "Country or currency," implying symbol search is supported. But the search JS matches on .country-name text and the currency full name — typing "€" or "£" most likely returns no results. Travelers often think in symbols when packing cash or checking receipts, so this is a real gap in the function the placeholder promises.

How Extend the search match to also check data-sym on each .country-block (already present in the HTML). One additional condition in the filter JS: || block.dataset.sym?.includes(q). No HTML changes needed.

Rate movement indicator vs. last month Feature

What Rates are refreshed monthly but the page shows no signal of direction — a traveler can't tell whether the Euro is stronger or weaker than when they last checked. A small ↑↓ arrow (or colored delta) next to each rate would immediately communicate budget pressure for upcoming trips.

How Have update_currency_rates.py save the previous month's rates to a Brain/scripts/currency_prev_rates.json cache before overwriting, then inject a data-prev-rate attribute alongside each data-rate on the country block. A small inline JS block computes the direction and renders the arrow. The cache file is Drive-only and never committed.

Inline styles on currency-code spans Look

What Every row in the index table contains <span style="color:var(--accent);">ARS</span> — an inline style on 71 spans. This is a site-wide CSS rule violation (SHARED CSS ONLY, added 2026-06-23) and means the color can't be updated from a shared CSS file. The spans are generated by build_currency.py's template, so the fix is in the script, not the HTML.

How Add a .cur-iso class to web-travel-style.css with color: var(--accent). Update the span template in build_currency.py to emit <span class="cur-iso"> instead of the inline style. Next time the page is structurally rebuilt (not a monthly refresh), this will be resolved.
