UX Issues — Logged for UX Routine (Not Fixed)
Low Outdated JS comment on EoA + preclearance filter behavior
The JS filter logic includes a comment: "GE/CLEAR/EoA pills are US-airport programs and don't apply to preclearance, so pills leave it alone." This comment was accurate when only YYZ/YVR had EoA badges. Now that all 14 preclearance airports carry EoA, the comment is misleading. The filter behavior itself is correct (EoA filter shows the intl section because all rows qualify), but the comment should be updated to reflect the changed reality.
Location: applyFilters() function comment above the #intl-section block.
Low Dead variable references: countEl and result
The filter script assigns var countEl = document.getElementById('count-num') and var result = document.getElementById('tt-result'). Neither id="count-num" nor id="tt-result" exists in the page HTML — both resolve to null and are never used after assignment. Suggests a planned result counter was stubbed out but never wired. Either remove the variables or add the elements.
Low Search autocomplete excludes international airports
The TVESearch.attach() block queries #airport-list .apt-row, which covers only the US domestic section. International preclearance airports (14 rows) are not included in the autocomplete item list. Typing "Dublin" or "Abu Dhabi" will not surface a suggestion in the dropdown, though the plain text search still filters the intl section correctly. Fix: also query #intl-section .apt-row when building the items array.
Low Inconsistent EoA label: filter pill vs. badge
The filter pill reads "Enrollment on Arrival" while the badge on each airport row reads "Enroll on Arrival". These are the same program but the labels differ by one character. Recommend standardizing on one form — CBP's official name is "Enrollment on Arrival." Update the badge text from "Enroll" to "Enrollment" to match the pill and official naming.
