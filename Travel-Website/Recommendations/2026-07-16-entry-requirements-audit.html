🎨 UX Handoff for weekly-ux-format-audit

File: Travel-Website/Trip-Essentials/Entry-Requirements.html — observe-only findings. Do not edit this file for visual reasons in this routine.

Color Status badges use hardcoded hex colors
Observed .status-live uses #e8f5e8 / #a0d8a0 / #2a6a2a; .status-coming uses #fff8e0 / #7a5800 / #e8c870. The CSS comment says "All colors via CSS variables — no hardcoded hex" but the badge implementations don't follow this.
Standard SHARED CSS ONLY rule (CLAUDE.md): never hardcode hex in pages — use CSS variables. check_css_duplication.py and check_no_hardcoded_styles.py enforce this.
Suggested fix Add --status-live-bg, --status-live-text, --status-live-bd, and coming equivalents to web-travel-style.css; reference them in the page .status-live / .status-coming rules.
Mobile UX Apply links column hidden on mobile — no CTA for e-visa applications
Observed The @media (max-width: 600px) block sets display: none on the 4th column (Apply links) of the e-visa table. Mobile users see Destination, Fee, and Valid/Stay — but no link to actually apply.
Impact Users most likely to need this page on mobile (planning from a phone) have no access to the apply links. This is the main CTA of the e-visa table.
Suggested fix Move the apply link into the Destination cell as a secondary line (.dest-note style) so it's always visible, then remove the mobile column-hide rule entirely. Or show a condensed link icon in a narrow 4th column on mobile.
Layout Search input appears above the alert banner and page description
Observed DOM order: h1 → search input → no-result msg → page description → alert banner → content. The search appears before users have seen context about what the page covers or the ETIAS delay alert.
Suggested fix Move the search input below the page description and alert banner, so the information hierarchy is: title → description → alert → search → content. The alert banner (now covering a delayed ETIAS) is important enough to be seen before the search.
CSS Apply button hover uses hardcoded terracotta border color
Observed .apply-btn:hover { border-color: #b85c2a; } — the terracotta color is hardcoded instead of using a CSS variable.
Standard Active/hover fill states should use #b85c2a or the terracotta gradient (Colors and Font Size.html § 11), but as a CSS variable, not a raw hex. Replace with var(--c-terracotta, #b85c2a) or similar.
Minor Inline style on search wrapper div
Observed The search wrapper uses style="display:flex;justify-content:center;margin:35px 0 18px;" as an inline style rather than a CSS class.
Suggested fix Extract to a class in the page <style> block: .er-search-wrap { display: flex; justify-content: center; margin: 35px 0 18px; }. Low priority but keeps the HTML clean.
