Recurring patterns & recommendations
also-on-this-site-pills wrapper missing (3 guides this cycle)
Affected: Cairo, Cancun + at least one prior session · Failure: TR-0b

Three guides in two audit cycles had the pills as direct children of #also-on-this-site without the required <div class="also-on-this-site-pills"> wrapper. The CSS grid layout that arranges the pill row depends on this wrapper. Without it, pills render unstyled and ungrouped. The pattern suggests older builds used a different wrapper class (pill-row) that was subsequently renamed.

A fleet-wide grep for class="pill-row" inside #also-on-this-site would identify any remaining affected guides before the next validator run catches them.

READ ABOUT injection pattern inconsistency (2 guides this cycle)
Affected: Big Island, Cape Cod · Failure: Read-About-Pages.html § 2

Two distinct failure modes in the same spec: Big Island used a bare script with parentNode.insertBefore (wrong structure entirely), while Cape Cod used the correct DOMContentLoaded + child-insertion structure but with the Unicode escape '›' instead of the literal '›' character in the button search loop. The validator's string check operates on source text, not runtime values.

The canonical pattern (as found in passing guides): document.addEventListener('DOMContentLoaded', fn) → query .overview-title → loop children for literal '›' → ot.insertBefore(a, nextBtn) with ot.appendChild(a) fallback. Any deviation in structure or character encoding fails.

Tours dangling tails — booking metadata appended after closing period (3 guides)
Affected: Boston, Bruges, Buenos Aires (3 entries) · Failure: Tours § 6

Five tour ↳ descriptions across three guides ended with fragments appended after a complete sentence: capacity info ("Max 16 per group with"), routing data ("and across Buenos Aires"), or booking attributes ("Vegan and dairy-free options available."). These appear to be copy-paste artifacts from source tour pages where booking attributes follow the main description.

The validator catches any sentence that ends with a word that looks like a booking attribute tail (capacity, availability, booking method). Worth checking builds more carefully before shipping — these are clean-up issues that don't require re-research.

Weekly Closures partial-category overlap — Buenos Aires
Affected: Buenos Aires · Rule: Weekly Closures - Extra Section.html § 2

The original rows split "Palaces" across three rows and "Government Sites" across two. The spec requires each category to appear in exactly one row. The fix required tracing actual closure days for each stop type: Teatro Colón (Theatres) → Sunday; museums → Monday; Casa Rosada (Government) → Tuesday; Palacio Barolo (Palaces) → Saturday. With each category mapped to one day, the four rows collapsed cleanly.

This is a content-accuracy issue, not just a formatting one — partial-category compound rows (e.g., "Government Sites & Palaces") can obscure that an individual venue actually closes on multiple days. The four-row result is cleaner but slightly lossy (Palaces actually close Sun/Tue/Sat; the guide now only shows Saturday for that category).

Getting Around structural issues — Bhutan, Boston
Affected: Bhutan (no extras-sub headings), Boston (leaking transit link) · Rule: Getting Around - Extra Section.html § 1b

Bhutan had two operator entries (DrukRide, DrukGo) as raw transit-boxes with no preceding extras-sub heading. The spec requires each operator to have its own heading. Boston had a more subtle issue: an MBTA Green Line link had leaked outside the Lyft transit-box and was floating as a bare <div> inside Getting Around — confirmed as a copy artifact since the guide has no tram motions in any day routes, so the tram entry was removed entirely rather than formatted.
