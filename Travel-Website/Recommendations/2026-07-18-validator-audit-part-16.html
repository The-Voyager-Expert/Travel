Fixes Applied

Fixed

Michelin section note (L804) — stale "no walk/ride" description removed

The section note at L804 of the Validator Index still described Style A as "Cuisine ≤80 chars / 📍 Address only — no hours, walk, ride, or closed rows". This was written at the Style A lock (2026-05-19) and never updated when the motion row was added in 2026-06-05. The 🚶/🚕 motion row is now required in §3b entries (Check D), so the section note was actively misleading.

Updated to: §3b format is Cuisine + 📍 + 🚶/🚕 motion row (required); 🏛 hours and 🚫 closed are §3a-only per Check J. §3a gets the review-link heading. Motion row added 2026-06-05.
Fixed

Michelin L817 — "🚶 walk, 🚕 ride" wrongly listed as banned

L817 said: "Entries must contain only ⭐ Name / Cuisine / 📍 Address rows — 🏛 hours, 🚶 walk, 🚕 ride, and 🚫 closed rows are banned."

This directly contradicted L818 in the same document, which states §3b entries carry a required 🚶 N min · 🚕 N min → hotel motion row. The code (Check D) makes 🚶/🚕 required — an entry missing 🚕 fails Check D. Only 🏛 and 🚫 are actually banned (Check J).

Updated L817 to: §3b bans 🏛 hours and 🚫 closed (Check J; §3a exempt). 🚶/🚕 are REQUIRED in §3b (not banned). CORE RULES §3b discrepancy noted — see Open Questions.
Fixed

Pickleball L794 — US-gate still described state-code detection (superseded 2026-07-06)

L794 described the conditional US gate as "validator detects US state code from title-address — Pattern A: STATE DDDDD · Pattern B: , STATE · Pattern C: City STATE." This was superseded on 2026-07-06 when the primary detection signal was switched from .title-address state-code parsing to .title-country == "United States". Hotel addresses use "Street · Neighborhood" format and never include a state code, so the old parser silently returned None for most US guides and the section gate quietly passed through without checking.

Updated L794: primary gate signal is .title-country == "United States" (always present, required per Hotel Banner §1). State-code patterns kept as informational hints in error messages only.

The current Michelin Restaurants - Extra Section.html §3b template explicitly shows 🏛 [Hours] and 🚫 Closed [Day] as format rows. Check J in the validator bans both in §3b entries with the comment "§3b format: ⭐ Name / Cuisine / 📍 Address / 🚶/🚕 → hotel; 🏛/🚫 are §3a in-hotel only."
