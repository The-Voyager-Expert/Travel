--track not defined in shared :root {} — silent transparent fallback on non-stats pages

web-travel-style.css line 729 has .bar-track { background: var(--track); }, but --track: #ece6dd was only defined inside Travel-Stats.html's inline <style> block. On any page using .bar-track that didn't define its own --track, the bar background resolved to transparent — a silent visual failure invisible in testing on the stats page.

Fix: Added --track: #ece6dd (light) and --track: #3a3530 (dark mode) to web-travel-style.css's shared :root {}. Pages that define their own value will still override it. Committed and pushed — live in commit 18f832d3f.

Variable also added to CSS-Architecture.md's reference table. To Do List item resolved.

Observations for Future Runs
Delta routing tier variables were never documented — pattern to watch

The 9 variables added to CSS-Architecture.md today all belong to a cluster of "specialized page" palette tokens (Delta, Weather, Pickleball, Cards-ATM). These are added to the shared :root {} to ensure they're available globally, but they're scoped in purpose to specific pages. This is correct per the Shared-CSS-Only rule, but it means new page-specific tokens can silently enter the shared stylesheet without being picked up in docs.

Mitigation: the sync table in CSS-Architecture.md now has a "Last synced" date; this routine should catch any new additions within its run cadence.
