Rules 555–559
Rule 555 HARD-FAIL ship
READ ABOUT injection missing margin-left:auto
The READ ABOUT injection <a> must set a.style.cssText containing margin-left:auto. Without it the link appears adjacent to the centered "TRIP OVERVIEW" text instead of right-aligned. 8 guides shipped without it on 2026-07-19: Bilbao, Cayman-Islands, Hilton-Head-Island, Foz-do-Iguaçu, Florida-Keys, Kauai, Marco-Island, Seychelles.

Authority: Read-About-Pages.html § 2
Rule 556 HARD-FAIL ship
READ ABOUT textContent must not contain ↗
The injected anchor text must be plain uppercase city name only (e.g. READ ABOUT HILTON HEAD ISLAND). The ↗ navigation indicator is exclusive to the hotel .title-address link. Hilton-Head-Island shipped with ↗ appended on 2026-07-19.

Authority: Read-About-Pages.html § 2
Rule 557 HARD-FAIL ship
Non-.title-address anchors must not contain ↗
The ↗ (U+2197) navigation indicator is exclusive to the hotel .title-address address link. Every other <a> whose anchor text contains ↗ fails — food delivery, Getting Around, shows, train booking. Rule 548 already bans ↗ on 📍 stop-row address links; Rule 557 catches all remaining contexts. Bilbao shipped 13 such links.

Authority: Icon Order and Format.html row 7 / Links.html
Rule 558 HARD-FAIL ship
.stop-photos must contain only <img>
No <div>, <figure>, <figcaption>, <span>, or other tag may appear inside .stop-photos. A non-<img> child becomes an unstyled flex child stretching to ~220px even when text is 1 line — creating a visible gap before the motion row. Photo credits belong as HTML comments. Bilbao shipped 16 .photo-credit divs inside .stop-photos.

Authority: guide-style.css STOP PHOTOS spec
Rule 559 HARD-FAIL session start
check_title_address_no_css_arrow — no .title-address a::after with border-top
guide-style.css must NOT carry a .title-address a::after rule block containing border-top. The CSS-drawn terracotta chevron arrow was removed on 2026-07-19; the native ↗️ emoji now renders directly from the hotel address HTML. Re-introducing the ::after block would render both a drawn geometric arrow AND the native emoji simultaneously. Right-padding was also reduced (32px → 11px; clearance for the drawn arrow is no longer needed). Checked at session start by brain_check.py; hard-fails immediately if the rule block reappears.
