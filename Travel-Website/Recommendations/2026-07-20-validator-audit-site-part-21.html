Validator Bug Fixed

FIXED build_days_data.py — display-name collision caused infinite false “Naples: 3 → 4” loop

Bug: Both Naples (Italy, 4 days, folder Naples/) and Naples (Florida, 3 days, folder Naples-Florida/) share the same dest-name = "Naples". DAYS_DATA keys by display name, so both guides mapped to the same "Naples" key. Guides process alphabetically — Italy first (derived["Naples"]=4), then Florida overwrites (derived["Naples"]=3). The change-detection recorded Italy’s update (3→4) but the final write produced 3 again. Result: audit_filters.py reported “Naples: 3 → 4” every single run and applied it each time, but it immediately reverted. An infinite false-positive loop.

Fix: Added duplicate display-name detection in build_days_data.py. When a second guide maps to a key already in derived, it logs a collision warning and keeps the higher day count (Italy’s 4 beats Florida’s 3). DAYS_DATA now correctly shows "Naples":4 and subsequent runs show no changes.

Owner action needed: To make DAYS_DATA filter work correctly for BOTH Naples guides, one card’s <span class="dest-name"> should be changed to Naples, FL or similar. Logged to To Do List.
