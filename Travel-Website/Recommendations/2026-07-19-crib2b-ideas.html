⭐ This Week's Featured Idea
Monthly Crowd Calendar
Interactive page — pick a month, see every destination ranked by expected tourist density

The site already answers "when is it warm?" (When to Go), "when is there a festival?" (Festival Finder), and "what's the climate?" (Climate Finder). It doesn't yet answer: "when is it quiet?" A crowd calendar fills that gap — the most underrated question serious travelers ask.

The feature: a 🗓️ month chip selector at the top, then a grid of destination cards each stamped with a four-tier crowd badge — Quiet / Moderate / Busy / Peak — plus a one-liner like "avoid July–August, sweet spot is May." Searchable by city name. Filterable by tier (show only Quiet destinations this month).

Implementation mirrors Festival-Finder.html exactly: curate a JSON dataset (one entry per guide city with 12 monthly crowd ratings), write build_crowd_calendar.py to generate the HTML, add a 🗓️ Crowd Calendar entry to the 🌤️ Weather toolbar group as a sibling of When to Go and Climate Finder. Sibling pills cross-link the three pages. The dataset can be seeded from tourism statistics, then refined editorially.

This complements (not duplicates) the 18 Jul OPEN_IDEAS "High season vs. shoulder season" proposal — that idea is an editorial article about seasons in general; this is an interactive tool with per-city, per-month data across the full guide fleet.

Dark Mode Toggle
Feature · toolbar.js · CSS custom properties

A persistent ☀️/🌙 icon button injected by toolbar.js into the right side of the navigation bar. Clicking it adds data-theme="dark" to <html> and stores the preference in localStorage. On page load, toolbar.js reads the stored preference before render to avoid flash.

The site's styles already use CSS custom properties (--bg, --text, --surface, --accent, etc.) across web-travel-style.css, guide-style.css, and mobile.css. Adding a [data-theme="dark"] :root {} block that overrides these tokens covers the entire site in one place. Respects prefers-color-scheme on first load as the default.

Distinct from the Accessibility Controls idea (crib3 top pick), which is about text size and contrast settings. Dark mode is purely a color-scheme preference, not an accessibility accommodation.

toolbar.js one-time change CSS token override block localStorage no per-page changes
✅
"I've Been" In-Guide Toggle
Feature · guide pages · Travel Tracker sync

A small ✓ Mark as Visited chip injected by toolbar.js into the Trip Overview section of every guide page. One tap writes the same tve-visited-{city} localStorage key that the Guides-Index been/want filter uses — so marking a guide "visited" from within the guide automatically updates the index card's status dot on next visit.

The chip renders differently based on current state: gray "Mark as Visited" when unvisited, terracotta "✓ Visited" when marked. Re-tapping unmarks. No navigation away required.

Distinct from all existing Travel Tracker ideas: the "What Next?" proposal (19 Jul base, F1) is about cross-guide recommendations, not marking state. The Shareable Wishlist (crib3, F4) is about future trips. This is about completing the in-guide experience loop so marking a trip doesn't require a separate app session.

toolbar.js injection localStorage sync guide pages only Guides-Index parity
🕐
"Open Right Now" Stop Filter
Feature · guide pages · client-side hours parsing

A 🕐 Open now filter pill in the Trip Overview day navigation that, when activated, dims or collapses stop blocks whose 🏛️ hours row doesn't include the current local time. Useful when a traveler is at the destination, guide open, deciding what to do at 3pm.

Implementation: client-side JS parses the 🏛️ row's text (e.g. "10am–6pm daily"), compares against new Date() in the destination's time zone (from the guide's Time Zones entry, or gracefully degraded to user locale). Stops with unparseable hours remain visible. State is not persisted — it resets on page reload. The pill appears only on guide pages and is injected by toolbar.js.

Hours parsing is feasible for the standardized formats already in the guide fleet (HH:MM–HH:MM daily, Mon–Fri HH:MM–HH:MM, etc.). Edge cases (seasonal, irregular) default to visible. No API calls — entirely offline-capable.

toolbar.js injection hours-row text parsing time zone aware no backend
🔀
Trip Twin — Find Your Match
Feature · Guides-Index · similarity scoring

A Find my match button on Guides-Index that helps first-time international travelers find a destination that feels familiar but is abroad. The user types their home city; a client-side similarity algorithm scores all guide destinations against it across three axes — climate profile (from CLIMATE_INLINE), cost tier (COST_DATA), and safety level (SAFETY_DATA) — and surfaces the 3 "closest to home" matches with a match score and a one-line reason.

All data is already embedded in Guides-Index as inline JS blocks. The similarity function runs entirely on page load with no network calls. Results appear in a compact tray below the search bar. For example: someone from Seattle (mild/temperate/moderate-cost) might match Oslo, Edinburgh, or Vancouver, BC.

Distinct from Help Me Choose (open-ended preference wizard), Compare mode (explicit A vs B), and Destination Records (flat rankings). Trip Twin is about personal similarity — it answers "where should I go first?" for the traveler who is uncertain.

Guides-Index addition CLIMATE_INLINE + COST_DATA + SAFETY_DATA cosine similarity no backend

New Guide: Ho Chi Minh City, Vietnam
New Guide · Southeast Asia · verified not in fleet

The fleet has Hanoi and Hoi An but no Saigon. Ho Chi Minh City is Southeast Asia's most electric metropolis: the War Remnants Museum, Bến Thành Market, the French-Quarter blocks of District 1, rooftop bars above the Saigon River, and the Cu Chi Tunnels as a half-day excursion.

Dense, walkable itinerary potential across Districts 1, 3, and 4. Strong tour ecosystem on Viator (Cu Chi Tunnels, street food night tours). Excellent Michelin Bib Gourmand presence (Cục Gạch Quán, Nhà Hàng Ngon). Pairs naturally as a companion to the Hoi An and Hanoi guides — a Vietnam arc.

New Guide: Havana, Cuba
New Guide · Caribbean · verified not in fleet

No Cuba destination exists in the fleet. Havana is unlike any other Caribbean entry: Old Havana (UNESCO World Heritage), the Malecón seafront, mid-century architecture, vintage American cars as the defining visual, live son and salsa at La Casa de la Música, Hemingway haunts (La Bodeguita del Medio, La Floridita). Entirely distinct from Aruba, Barbados, Cayman, and the other fleet islands.

Logistical note: US travelers need a general license (tourism category); the guide would note this under Heads Up. Entry requirements via CLAUDE.md routing. No standard Viator coverage — local operators (Intrepid, G Adventures) and Atlas Obscura sourcing for stops.

Cuba (new country) 3 day guide US license note required

Overtourism Survival Guide
New Trip-Essentials Page · editorial + tactics

An editorial page with actionable tactics for visiting the fleet's ten most-crowded marquee stops — Trevi Fountain (Rome), Cinque Terre villages, Santorini cliff-edge, Dubrovnik Old Town, Eiffel Tower base, Prague Charles Bridge, Amsterdam canal bridges, Kyoto Fushimi Inari, the Colosseum, Stonehenge. Per-entry content: the best window (exact hour), timed-entry booking link, shoulder-month recommendation, and one less-visited alternative nearby.

Lives as a standalone Trip-Essentials page. Cross-links to the relevant guides. Reachable from the Best-Of index as a "Planning & Tactics" entry, or from the relevant guides' Heads Up sections. Not an editorial opinion piece — entirely tactical and link-rich.

Distinct from all existing OPEN_IDEAS: this is about navigating crowds at specific known stops, not about avoiding tourist scams (Common Travel Scams, open), not about seasonal planning (High season vs. shoulder, open), and not an arrival guide.

Best Destinations for Rainy Days — New Best-of Page
New Best-of Page · 33-page collection addition

A new entry in the Best-of collection — joining the 33 existing pages — curating destinations whose character is enhanced by rain: Amsterdam canal cafés under a grey sky, Bergen fjord drizzle, Kyoto temple paths in mist, Paris arcades and covered passages, Prague cobblestones reflecting lamplight, Vienna Kaffeehäuser.

Implementation: build_showcase_cards.py from a JSON entry file, same pipeline as all other Best-of pages. Roughly 20–25 entries across the fleet.
