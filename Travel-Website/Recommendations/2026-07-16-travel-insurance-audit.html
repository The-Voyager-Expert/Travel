Tips — Key Things to Know

Added
New tip: CFAR cancel deadline (48–72 hrs before departure)

Most Cancel For Any Reason policies require canceling at least 48 hours before departure (some require 72 hours). Travelers often assume CFAR works up to the moment of departure — it does not. Confirmed against Squaremouth and InsureMyTrip 2026 CFAR guidance. Notable gap in the existing tip set.

All 6 existing tips confirmed accurate
  • Buy within 14–21 days of first deposit ✓
  • Covered reason definition (illness, death, jury duty, severe weather — not work conflicts) ✓
  • Must pay with card to activate card coverage ✓
  • Medicare and most US health plans stop at the border ✓
  • Activity exclusions (skiing, diving, motorbike require World Nomads or rider) ✓
  • Don't buy the cruise line's insurance ✓

Suggestions — look, function & features

Look
Transpose the credit card comparison table — column per card, row per coverage type

The current table has cards as rows, which means comparing a single coverage type (e.g., Trip Delay) across all 5 cards requires reading across a wide row. Transposing so each card is a column and each coverage type is a row lets a user scan down one column to evaluate a specific card, or scan across one row to compare all cards on a single attribute. This is how most credit card comparison sites lay out this data — it maps naturally to the "I have this card, what do I have?" use case and would be especially valuable on mobile.

Look
Add category chips to provider tags — scannable without reading descriptions

Each `.prov-tag` subtitle is plain text. Adding a small colored category chip (e.g., 🏥 Medical, 🏔️ Adventure, 👨‍👩‍👧 Families, 📅 Annual) beside the tag would let a scanner find the right provider in 2 seconds without reading 8 descriptions in full. The chip encodes the same niche as the tag, just glanceable. Could double as filter anchors (see Feature suggestion below).

Function
Add anchor jump nav at the top — four sections, no visible way to jump between them

The page has four distinct content sections — Credit Cards, Coverage Guidance, Providers, Tips — but no anchor navigation. A user who arrives knowing they want to compare providers has to scroll past the credit card table to reach that section. A compact pill row at the top (Cards · Guidance · Providers · Tips) using `.also-on-this-site-pill` styling would let users jump directly. On mobile this is especially useful: the page is long and the sections aren't visually distinct enough to orient quickly by scrolling.

Function
Card coverage quick-check — select your card, see a summary of what you already have

A common arrival scenario: "I have a Sapphire Reserve — do I need to buy separate insurance?" The page answers this but requires finding the right card row in the table and reading across it. A `<select>` dropdown (or 5 pill buttons) of the covered cards — on selection, highlights that card's row and surfaces a compact 3-line summary ("Trip cancel ✓ $10K/person · Medical ✓ · Evacuation ✓ $100K") — would answer the question in one tap. Pure JS, no backend, uses the data already on the page.

Feature
Trip cost estimator — enter trip value, see the insurance cost range in dollars

The guidance section already says standalone insurance costs 4–10% of trip value, but this is abstract until you have a number. A simple inline estimator (one number input for trip cost → one output line: "A $6,000 trip → ~$240–$600 in premiums") makes the cost concrete and helps users decide whether to look at standalone coverage at all. Pure JS calculation using the 4–10% range already on the page. Could sit directly below the coverage guidance cards.

Feature
Provider filter chips — filter the 8 providers by niche without scrolling

At 8 providers (and potentially growing), the provider section is getting long. Filter chips (All · Medical Focus · Adventure · Families · Annual Plans) built on `data-category` attributes on each `.provider-row` would let users surface only the providers relevant to their trip type. Same pattern as the Guides-Index filter chips — hide non-matching rows, show all on "All". The category chips naturally pair with the Look suggestion above (chips on the tags) — the same data attribute serves both the chip label and the filter.
