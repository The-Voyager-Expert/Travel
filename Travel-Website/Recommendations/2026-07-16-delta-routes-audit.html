UX observations — logged to handoff (2026-07-16-delta-routes-ux-handoff.html)
👀 `:root {}` defined in inline page CSS on both pages
Both pages define tier color vars (--c0, --c1, --c2 families) in an in-page :root {} block. Site rule bans :root {} in pages — these belong in web-travel-style.css.
👀 `#fdf3e0` (banned beige) used as tier-2 background on both pages
--c2bg: #fdf3e0 powers the amber tier pill, badge, and card backgrounds. #fdf3e0 is listed in DriftyCat as a banned beige pill/button background.
👀 Hardcoded hex colors throughout inline CSS on both pages
Multiple inline property values use bare hex: placeholder color #A8895A, focus border #c8b99a, tag backgrounds #f8e8e8/#7a1010, hub-pill #f8ece8/#e8c0b0. Site rule: use CSS variables, never hardcode.
👀 Active tier-pill state uses dark shadow, not terracotta
.tier-pill.active adds bold + dark box-shadow ring but no background change. These pills are interactive filters — active state should use terracotta #b85c2a like .nav-link.active on the same page.
👀 "Updated July 2026" stamp floats outside header block (SEA page only)
The stamp sits in a standalone .wrap div after the header div closes, creating layout inconsistency vs the Full page. Check whether SEA page's toolbar-mount has a data-updated attribute; if so, the manual stamp is redundant.
Suggested improvements — look, function, and features
1 · Suspended routes need a visual badge, not just text
Routes like SEA–CUN (suspended) and ATL–TLV (suspended) show suspension only in dest-meta text — easy to miss when scanning a card grid. Adding a .tag.suspended span (red/amber styling, like .tag.d1 already on both pages) would make suspended routes immediately visible without reading the meta line.

Spec: Add .tag.suspended { background: #fce8e8; color: #7a1010; } to shared CSS (same family as .tag.d1). Add <span class="tag suspended">Suspended</span> to any card whose dest-meta begins with "Suspended" or contains "Resumes".
2 · Add a Status quick-filter chip row to both pages
Both pages filter by tier (Nonstop / 1-connection / 2-connection) but have no way to filter by status. A "Status" chip row — Year-round · Seasonal · New · Suspended — would let users instantly surface all suspended routes (useful during disruption periods) or all routes still in a launch "new" state. The JS pattern for the tier filter is already on both pages and is trivially extensible.

Spec: Add a second chip row below the tier-legend. Each chip sets a data-status attribute filter matched against the card's tag spans. Can be combined with the tier filter, or mutually exclusive.
3 · Guide cross-links on the Full page (missing; present on SEA page)
The SEA page injects a 📖 Guide link on any card whose city has a matching guide, via a GUIDES JS map and .has-guide hover treatment. The Full page has identical card markup but no guide cross-link injection — users looking up ATL routes to Paris, Rome, or Amsterdam see no path to the guide.

Spec: Copy the GUIDES map object and findGuide() injection script from the SEA page into the Full page. No HTML change needed — .dest-city elements are already there for the script to read.
4 · Hub-strength lines: IATA codes as scannable chips, not prose
Hub-strength lines are dense prose: "LHR, AMS, CDG (seasonal), FCO (seasonal summer), BCN (seasonal summer)…" Wrapping each IATA code in a small styled span — matching the .hub-pill style already defined on the Full page — would make the lines visually scannable without changing any content or data.

Spec: Wrap each 3-letter IATA code in hub-strength lines with <span class="hub-pill">LHR</span>. The .hub-pill CSS is already present on the Full page. On the SEA page, define the same .hub-pill class and wrap the Connection Guide hub labels.
5 · Seasonal calendar strip for date-bounded routes
Routes with specific season windows (SEA–BCN May 7–Oct 22, SEA–FCO May 6–Oct 23, SEA–CUN resumes Nov 9) express their windows only as text in dest-meta. A compact 12-month calendar strip — shaded for the operating months — would make seasonal coverage immediately visual and far more useful for trip planning.

Spec: For cards with a data-season-start / data-season-end attribute (month number 1–12), inject a 12-cell strip via JS where operating months are shaded in the card's tier color and off-season cells are muted. Strip should be narrow enough to fit inside the existing card without changing its grid height significantly (aim for 20px tall).
6 · Flight time estimates on Full page nonstop cards
The SEA page injects approximate nonstop flight times (⏱ ~10h nonstop) on tier-0 cards via a FT JS lookup keyed by IATA code. The Full page has no equivalent — a user looking at ATL–JNB or JFK–FCO sees no flight time. Adding flight time injection on the Full page for international nonstops at each hub would give meaningful trip-planning context.

Spec: Add a FT lookup object covering major international nonstops at each hub (ATL, JFK, BOS, DTW, MSP, LAX, SLC). Inject the .flt span where FT[code] exists. Scope to international nonstops only — skip domestic. Block times sourced from delta.com or verified aviation data.
