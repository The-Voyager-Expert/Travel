CORE RULES spec bug
Time Zones fragment spec was wrong — Guide Structure.html §2 said "use folder-name hyphen form (e.g. #Hilton-Head-Island)" for all cities. Audit of Hong Kong revealed this is only valid for cities that have an explicit guide:'...' field in Time-Zones.html CITIES. Cities without that field (including Hong Kong) only match on c.name — hash #Hong-Kong fails; #Hong Kong is the only valid form.
Root cause: spec was written from a multi-word-city example (Hilton Head Island has guide:'Hilton-Head-Island') and incorrectly generalized. Fix: spec corrected to "use city name from Time-Zones.html CITIES array; cities with a guide: field also accept folder-name form as alternative." Hong Kong guide was already correct (#Hong Kong); no guide HTML change needed.
