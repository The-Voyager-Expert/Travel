P2 New · confirmed
brain_check.py css-duplication warning always shows "across 0 pages"

Root cause: brain_check.py line 2418 searches for "Total pages affected:\s*(\d+)" in check_css_duplication.py's stdout. But the script prints "Total pages with inline styles: 128" — different label. The regex never matches, mp is None, pages defaults to 0, and every session start warns: "255 duplicated CSS selectors across 0 pages." This makes the warning look like a false alarm (0 pages affected?) and has been masking the real severity since the Guides-Index relocation.

Action: In Brain/scripts/brain_check.py line 2418, change the regex from r"Total pages affected:\s*(\d+)" to r"Total pages with inline styles:\s*(\d+)". One character change; the warning will then correctly read "255 duplicated CSS selectors across 128 pages."
